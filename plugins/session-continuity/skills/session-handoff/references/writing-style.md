# Handoff Writing Style

Handoffs help a future session resume without re-deriving context. Every word should pay rent.

## Rules

- Prefer bullets over paragraphs.
- Lead with outcomes, decisions, and concrete changes.
- Use `path:line`, commit SHAs, or issue IDs instead of restating code.
- Keep `Summary` to 1-3 sentences.
- Put details in `Work done`; put unresolved action in `Open questions / next steps`.
- Do not repeat the same fact across sections.
- Preserve a short code/data snippet only when it is the substance of the finding.

## Cut

- Marketing adjectives: "seamless", "powerful", "robust", "elegant", "comprehensive".
- Filler and hedges: "in order to", "it should be noted", "please note", "it is important to".
- Vague process verbs without outcome: "explored", "investigated", "discussed".
- Narrating the obvious: "we made a commit".
- Editorial praise.

Use `will` only for confirmed future work, `should` for recommendations, and `might` for speculation.
